    }

    }
}

## 🔧 技术特点

### 1. 事务支持
- 所有更新操作在同一个事务中执行
- 任何一条记录失败将全部回滚
- 保证数据一致性

### 2. 动态字段更新
- 只更新提供的字段
- 未提供的字段保持不变
- 灵活适应各种业务场景

### 3. 数据验证
- 前端验证：建议在前端验证房间和床位的存在性
- 后端验证：验证人员 ID 的有效性
- 数据库约束：外键约束保证数据完整性

### 4. 性能优化
- 批量操作减少数据库交互次数
- 相比循环调用单次更新，性能提升显著
- 适合大批量数据更新

## 📈 性能对比

### 单次更新 vs 批量更新

假设有 100 个人员需要更新：

**单次更新方式**:
```java
for (Person person : persons) {
    personService.updatePerson(person);  // 100 次数据库交互
}
```
- 数据库交互次数：100 次
- 耗时：约 500-1000ms

**批量更新方式**:
```java
personService.batchUpdateRoomInfo(persons);  // 1 次数据库交互
```
- 数据库交互次数：1 次
- 耗时：约 50-100ms

**性能提升**: 约 5-10 倍
