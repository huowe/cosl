批量更新人员房间信息 - 使用示例

📋 说明

本页面展示了如何使用批量更新人员房间信息的 API 接口。以下是多个实际场景的示例代码和演示。

接口基本信息

属性 值
接口地址 POST /cosl/api/person/batchUpdateRoomInfo
Content-Type application/json
所需权限 system:person:edit

示例 1: 批量分配房间给新上船人员

场景描述：5 名新船员上船，需要为他们分配房间和床位

请求 JSON 数据：

[ { "id": 1, "roomId": 101, "bedId": 201, "status": "ONBOARD" }, { "id": 2, "roomId": 101, "bedId": 202, "status": "ONBOARD" }, { "id": 3, "roomId": 102, "bedId": 203, "status": "ONBOARD" }, { "id": 4, "roomId": 102, "bedId": 204, "status": "ONBOARD" }, { "id": 5, "roomId": 103, "bedId": 205, "status": "ONBOARD" } ]

模拟响应结果：

{ "msg": "成功更新 5 条记录", "code": 0 }

示例 2: 批量调整房间

场景描述：因房间维修，需要将 A101 房间的人员调整到 A201 房间

请求 JSON 数据：

[ { "id": 1, "roomId": 201, "bedId": 301 }, { "id": 2, "roomId": 201, "bedId": 302 } ]

模拟响应结果：

{ "msg": "成功更新 2 条记录", "code": 0 }

示例 3: 批量更新人员状态

场景描述：一批人员转船，需要更新状态为 TRANSFER

请求 JSON 数据：

[ { "id": 1, "status": "TRANSFER" }, { "id": 2, "status": "TRANSFER" }, { "id": 3, "status": "TRANSFER" } ]

说明：只更新状态字段，不改变房间和床位信息

模拟响应结果：

{ "msg": "成功更新 3 条记录", "code": 0 }

示例 4: JavaScript AJAX 调用示例

以下是使用原生 JavaScript 发起批量更新请求的代码：

// 批量更新人员房间信息 async function batchUpdateRoomInfo(persons) { try { const response = await fetch('/cosl/api/person/batchUpdateRoomInfo', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify(persons) }); const result = await response.json(); if (result.code === 0) { console.log('批量更新成功:', result.msg); alert('批量更新成功！'); return true; } else { console.error('批量更新失败:', result.msg); alert('批量更新失败：' + result.msg); return false; } } catch (error) { console.error('请求错误:', error); alert('网络错误：' + error.message); return false; } } // 使用示例 const personsToUpdate = [ { id: 1, roomId: 101, bedId: 201, status: 'ONBOARD' }, { id: 2, roomId: 101, bedId: 202, status: 'ONBOARD' }, { id: 3, roomId: 102, bedId: 203, status: 'ONBOARD' } ]; // 调用函数 batchUpdateRoomInfo(personsToUpdate);

示例 5: Vue.js 调用示例

以下是使用 Vue.js + Axios 发起批量更新请求的代码：

<template> <div> <el-button type="primary" @click="handleBatchAssign">批量分配房间</el-button> </div> </template> <script> export default { data() { return { persons: [ { id: 1, roomId: 101, bedId: 201 }, { id: 2, roomId: 101, bedId: 202 }, { id: 3, roomId: 102, bedId: 203 } ] }; }, methods: { async handleBatchAssign() { try { const { data } = await this.$axios.post( '/cosl/api/person/batchUpdateRoomInfo', this.persons ); if (data.code === 0) { this.$message.success(data.msg); } else { this.$message.error(data.msg); } } catch (error) { this.$message.error('批量更新失败：' + error.message); } } } }; </script>

示例 6: React 调用示例

以下是使用 React + Axios 发起批量更新请求的代码：

import React, { useState } from 'react'; import axios from 'axios'; import { Button, message } from 'antd'; const PersonBatchUpdate = () => { const [loading, setLoading] = useState(false); const handleBatchUpdate = async () => { const persons = [ { id: 1, roomId: 101, bedId: 201, status: 'ONBOARD' }, { id: 2, roomId: 101, bedId: 202, status: 'ONBOARD' }, { id: 3, roomId: 102, bedId: 203, status: 'ONBOARD' } ]; setLoading(true); try { const response = await axios.post( '/cosl/api/person/batchUpdateRoomInfo', persons ); if (response.data.code === 0) { message.success(response.data.msg); } else { message.error(response.data.msg); } } catch (error) { message.error('批量更新失败：' + error.message); } finally { setLoading(false); } }; return ( <Button type="primary" onClick={handleBatchUpdate} loading={loading} > 批量更新房间 </Button> ); }; export default PersonBatchUpdate;
⚠️ 注意事项
  • 必填字段：每个人员对象必须包含 id 字段
  • 可选字段：roomId、bedId、lifeboatId、status 都是可选的，只提供需要更新的字段
  • 事务性：所有更新操作在同一个事务中执行，任何一条记录失败将全部回滚
  • 权限要求：需要 system:person:edit 权限
  • 数据验证：建议在前端先验证房间和床位的存在性和可用性

常见错误处理

错误信息 原因 解决方案
批量更新数据不能为空 请求体为空数组或 null 确保请求体包含至少一个人员对象
人员 ID 不能为空 人员对象缺少 id 字段 检查每个人员对象都包含有效的 id
401 Unauthorized 未登录或 token 过期 重新登录获取 token
403 Forbidden 没有编辑权限 联系管理员分配 system:person:edit 权限
