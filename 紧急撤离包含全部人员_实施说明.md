## 🔧 技术实现细节

### 核心业务逻辑

```java
@Override
@Transactional
public int insertEvacuationWithAllPersons(Evacuation evacuation)
{
    // 1. 先插入撤离事件
    int result = evacuationMapper.insertEvacuation(evacuation);
    
    if (result > 0 && evacuation.getId() != null)
    {
        // 2. 查询当前平台下的所有人员
        Person queryPerson = new Person();
        queryPerson.setPlatformNo(evacuation.getPlatformNo());
        List<Person> allPersons = personMapper.selectPersonList(queryPerson);
        
        // 3. 为每个人员创建撤离记录
        if (allPersons != null && !allPersons.isEmpty())
        {
            for (Person person : allPersons)
            {
                EvacuationRecord record = new EvacuationRecord();
                record.setEvacuationId(evacuation.getId());
                record.setPersonId(person.getId());
                record.setPersonName(person.getName());
                record.setStatus("PENDING"); // 待撤离状态
                
                evacuationMapper.insertEvacuationRecord(record);
            }
            
            // 4. 更新预计撤离人数
            Evacuation updateEvacuation = new Evacuation();
            updateEvacuation.setId(evacuation.getId());
            updateEvacuation.setExpectedCount(allPersons.size());
            evacuationMapper.updateEvacuation(updateEvacuation);
        }
    }
    
    return result;
}
```

### 事务控制

- 使用 `@Transactional` 注解确保整个操作的原子性
- 如果任何步骤失败，所有数据库操作都会回滚
- 保证数据一致性

## 🗄️ 数据库影响

### 涉及的表

1. **evacuation（紧急撤离事件表）**
   - 插入一条新记录
   - 自动填充：id, platform_no, name, type, alarm_level, status, create_time
   - 后续更新：expected_count（预计撤离人数）

2. **evacuation_record（撤离人员记录表）**
   - 为每个人员插入一条记录
   - 字段：evacuation_id, person_id, person_name, status='PENDING'

### 人员状态说明

- **PENDING**: 待撤离（初始状态）
- **EVACUATED**: 已撤离
- **MISSING**: 失踪
- **INJURED**: 受伤

## ⚠️ 注意事项

1. **平台隔离**
   - 只会添加当前登录用户所属平台的人员
   - 通过 `PlatformContext.getPlatformNo()` 获取

2. **性能考虑**
   - 如果人员数量很大（如超过1000人），可能需要优化
   - 建议使用批量插入而非循环单条插入

3. **权限控制**
   - 需要 `system:evacuation:add` 权限
   - 使用 Shiro 进行权限验证

4. **日志记录**
   - 使用 `@Log` 注解记录操作日志
   - 便于审计和问题追踪

5. **重复创建**
   - 每次调用都会创建新的撤离事件
   - 不会检查是否存在重复的撤离事件

## 🔍 故障排查

### 问题1：创建失败，提示权限不足

**原因：** 当前用户没有 `system:evacuation:add` 权限

**解决：** 
- 检查用户角色配置
- 在系统中为该角色添加相应权限

### 问题2：人员记录未创建

**原因：** 当前平台下没有人员数据

**解决：**
- 检查 person 表中是否有该平台的记录
- 确认 platform_no 字段是否正确设置

### 问题3：事务回滚

**原因：** 某个步骤出现异常

**解决：**
- 查看日志文件，定位具体错误
- 检查数据库连接和表结构
