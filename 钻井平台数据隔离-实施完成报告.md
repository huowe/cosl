## 🎯 核心API接口

### 平台管理API

| 接口 | 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|------|
| 获取平台列表 | GET | `/cosl/api/platform/list` | 获取所有启用的平台 | 无 |
| 切换平台 | POST | `/cosl/api/platform/switch` | 切换到指定平台 | 无 |
| 获取当前平台 | GET | `/cosl/api/platform/current` | 获取当前选择的平台 | 无 |
| 平台配置列表 | POST | `/system/platform/list` | 平台配置管理列表 | system:platform:list |
| 新增平台 | POST | `/system/platform/add` | 新增平台配置 | system:platform:add |
| 修改平台 | POST | `/system/platform/edit` | 修改平台配置 | system:platform:edit |
| 删除平台 | POST | `/system/platform/remove` | 删除平台配置 | system:platform:remove |

### 使用示例

```bash
# 1. 获取所有平台
curl http://localhost:8080/cosl/api/platform/list

# 2. 切换到PLATFORM-A01
curl -X POST http://localhost:8080/cosl/api/platform/switch \
  -H "Content-Type: application/json" \
  -d '{"platformNo":"PLATFORM-A01"}'

# 3. 查询人员(自动按平台过滤)
curl http://localhost:8080/cosl/api/person/list \
  -H "X-Platform-No: PLATFORM-A01"
```

## 🔧 技术实现要点

### 1. 数据隔离机制

- **查询隔离**: SQL中添加 `platform_no` 过滤条件
- **新增隔离**: Service层自动从上下文获取并设置 `platform_no`
- **线程安全**: 使用 `ThreadLocal` 存储当前平台号
- **自动清理**: 请求完成后自动清除上下文,避免内存泄漏

### 2. 拦截器工作流程

```
用户请求 
  ↓
PlatformInterceptor拦截
  ↓
从Header(X-Platform-No)或参数获取platformNo
  ↓
设置到PlatformContext(ThreadLocal)
  ↓
业务逻辑处理(自动使用platformNo过滤)
  ↓
请求完成,清除PlatformContext
  ↓
返回响应
```

### 3. 前端集成方式

```javascript
// 在axios请求拦截器中自动添加平台号
axios.interceptors.request.use(config => {
  const platformNo = localStorage.getItem('currentPlatformNo')
  if (platformNo) {
    config.headers['X-Platform-No'] = platformNo
  }
  return config
})
```
