# 钻井平台数据隔离功能 - 快速开始指南

## 📋 概述

本系统已成功实现多钻井平台数据隔离功能。各钻井平台的数据完全独立,用户可通过切换平台号查看不同平台的数据。

## ✅ 已完成的工作

### 1. 数据库层面
- ✅ 创建 `platform_config` 表(钻井平台配置表)
- ✅ 为16个业务表添加 `platform_no` 字段和索引
- ✅ 插入3个示例平台数据
- ✅ 迁移脚本:`sql/platform_config.sql`

### 2. 后端核心组件
- ✅ `PlatformConfig` 实体类 - 平台配置模型
- ✅ `PlatformConfigMapper` - 平台数据访问层
- ✅ `IPlatformConfigService` / `PlatformConfigServiceImpl` - 平台业务逻辑层
- ✅ `PlatformConfigController` - 平台管理控制器

- ✅ `PlatformInterceptor` - 平台切换拦截器

- ✅ 拦截路径: `/cosl/api/**`
- ✅ 从请求头 `X-Platform-No` 或参数 `platformNo` 获取平台号

### 4. 示例修改

- ✅ `PersonMapper.xml` 添加平台过滤条件

## 🚀 快速开始

### 步骤1: 执行数据库迁移

```bash
# 在MySQL命令行中执行
mysql -u root -p cosl < sql/platform_config.sql
```

或直接导入 `sql/platform_config.sql` 文件。

### 步骤2: 重启应用

重新启动 Spring Boot 应用,确保新代码生效。

### 步骤3: 测试API

#### 3.1 获取平台列表
```bash
curl http://localhost:8080/cosl/api/platform/list
```

#### 3.2 切换平台
```bash
curl -X POST http://localhost:8080/cosl/api/platform/switch \
  -H "Content-Type: application/json" \
  -d '{"platformNo":"PLATFORM-A01"}'
```

#### 3.3 查询人员(携带平台号)
```bash
curl http://localhost:8080/cosl/api/person/list \
  -H "X-Platform-No: PLATFORM-A01"
```

### 步骤4: 前端集成

参考 `钻井平台数据隔离功能使用说明.md` 中的前端集成示例。

## 💡 使用建议

### 1. Service层自动注入平台号

在所有新增方法中,自动从上下文获取平台号:

```java
@Override
public int insertXXX(XXX xxx)
{
    // 从上下文获取当前平台号
    String platformNo = PlatformContext.getPlatformNo();
    if (StringUtils.isNotEmpty(platformNo))
    {
        xxx.setPlatformNo(platformNo);
    }
    xxx.setCreateBy(getLoginName());
    return xxxMapper.insertXXX(xxx);
}
```

### 2. 前端统一处理

在请求拦截器中自动添加平台号到请求头:

```javascript
// axios 请求拦截器
axios.interceptors.request.use(config => {
  const platformNo = localStorage.getItem('currentPlatformNo')
  if (platformNo) {
    config.headers['X-Platform-No'] = platformNo
  }
  return config
})
```

### 3. 平台选择器位置

建议在以下位置添加平台选择器:
- 顶部导航栏右侧
- 用户头像下拉菜单中
- 首页Dashboard显著位置

## ⚠️ 注意事项

1. **首次使用**: 执行SQL后,所有现有数据的 `platform_no` 都被设置为 `PLATFORM-A01`
2. **默认平台**: 如果未设置平台号,查询可能返回空数据
3. **权限控制**: 目前所有用户可访问所有平台,如需限制需额外开发
4. **性能优化**: `platform_no` 字段已添加索引,查询性能有保障
