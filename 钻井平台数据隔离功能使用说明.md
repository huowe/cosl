# 钻井平台数据隔离功能 - 使用说明

## 功能概述

本系统已实现多钻井平台数据隔离功能，支持：
- 创建和管理多个钻井平台配置
- 各平台数据完全隔离（人员、房间、救生艇、摄像头、演练等）
- 通过切换平台号查看不同平台的数据
- 自动根据当前选择的平台过滤数据

## 一、数据库迁移

### 1. 执行SQL脚本

运行 `sql/platform_config.sql` 脚本，该脚本会：
- 创建 `platform_config` 表（钻井平台配置表）
- 为所有业务表添加 `platform_no` 字段
- 插入3个示例平台数据
- 将现有数据的平台号设置为默认值 `PLATFORM-A01`

```bash
# 在MySQL中执行
mysql -u root -p cosl < sql/platform_config.sql
```

### 2. 验证数据

```sql
-- 查看平台配置
SELECT * FROM platform_config;

-- 查看人员的平台号
SELECT id, name, platform_no FROM person;
```

## 三、前端集成示例

### 1. Vue.js 中使用

```javascript
// src/api/platform.js
import request from '@/utils/request'

// 获取平台列表
export function getPlatformList() {
  return request({
    url: '/cosl/api/platform/list',
    method: 'get'
  })
}

// 切换平台
export function switchPlatform(platformNo) {
  return request({
    url: '/cosl/api/platform/switch',
    method: 'post',
    data: { platformNo }
  })
}

// 设置请求拦截器，自动添加平台号到请求头
request.interceptors.request.use(config => {
  const platformNo = localStorage.getItem('currentPlatformNo')
  if (platformNo) {
    config.headers['X-Platform-No'] = platformNo
  }
  return config
})
```

### 2. 平台选择组件

```vue
<template>
  <el-select v-model="currentPlatform" @change="handlePlatformChange" placeholder="选择钻井平台">
    <el-option
      v-for="platform in platforms"
      :key="platform.platformNo"
      :label="platform.platformName"
      :value="platform.platformNo">
      <span>{{ platform.platformName }}</span>
      <span style="color: #8492a6; font-size: 13px">{{ platform.location }}</span>
    </el-option>
  </el-select>
</template>

<script>
import { getPlatformList, switchPlatform } from '@/api/platform'

export default {
  data() {
    return {
      platforms: [],
      currentPlatform: ''
    }
  },
  created() {
    this.loadPlatforms()
    // 从本地存储恢复上次选择的平台
    this.currentPlatform = localStorage.getItem('currentPlatformNo') || ''
  },
  methods: {
    async loadPlatforms() {
      const response = await getPlatformList()
      this.platforms = response.data
    },
    async handlePlatformChange(platformNo) {
      try {
        await switchPlatform(platformNo)
        // 保存到本地存储
        localStorage.setItem('currentPlatformNo', platformNo)
        // 刷新当前页面数据
        this.$message.success('平台切换成功')
        this.$emit('platform-changed', platformNo)
      } catch (error) {
        this.$message.error('平台切换失败')
      }
    }
  }
}
</script>
```

### 3. 在所有API请求中自动携带平台号

```javascript
// src/utils/request.js
import axios from 'axios'

const service = axios.create({
  baseURL: process.env.VUE_APP_BASE_API,
  timeout: 5000
})

// 请求拦截器
service.interceptors.request.use(
  config => {
    // 从localStorage获取当前平台号
    const platformNo = localStorage.getItem('currentPlatformNo')
    if (platformNo) {
      // 添加到请求头
      config.headers['X-Platform-No'] = platformNo
    }
    return config
  },
  error => {
    return Promise.reject(error)
  }
)

export default service
```

## 四、后端开发指南

### 1. 实体类添加 platform_no 字段

所有业务实体类都已添加 `platformNo` 字段，例如：

```java
public class Person extends BaseEntity
{
    /** 钻井平台编号 */
    private String platformNo;

    public String getPlatformNo()
    {
        return platformNo;
    }

    public void setPlatformNo(String platformNo)
    {
        this.platformNo = platformNo;
    }
}
```

### 2. Mapper XML 添加平台过滤

在查询SQL中添加平台号过滤条件：

```xml
<select id="selectPersonList" parameterType="Person" resultMap="PersonResult">
    <include refid="selectPersonVo"/>
    <where>
        deleted = 0
        <!-- 平台号过滤 -->
        <if test="platformNo != null and platformNo != ''">
            AND platform_no = #{platformNo}
        </if>
        <!-- 其他查询条件 -->
        <if test="name != null and name != ''">
            AND name like concat('%', #{name}, '%')
        </if>
    </where>
</select>
```

### 3. Service层自动注入平台号

在Service层新增数据时，自动设置平台号：

```java
@Override
public int insertPerson(Person person)
{
    // 从上下文获取当前平台号
    String platformNo = PlatformContext.getPlatformNo();
    if (StringUtils.isNotEmpty(platformNo))
    {
        person.setPlatformNo(platformNo);
    }
    person.setCreateBy(getLoginName());
    return personMapper.insertPerson(person);
}
```

## 五、数据隔离说明

### 1. 隔离范围

以下数据按平台隔离：
- ✅ 人员信息 (person)
- ✅ 房间配置 (room_config)
- ✅ 房间床位 (room_bed)
- ✅ 楼层配置 (floor_config)
- ✅ 摄像头 (camera)
- ✅ 救生艇配置 (lifeboat_config)
- ✅ 救生艇座位 (lifeboat_seat)
- ✅ 救生艇人员关联 (lifeboat_person)
- ✅ 撤离点 (evacuation_point)
- ✅ 演练 (drill)
- ✅ 演练人员关联 (drill_person)
- ✅ 撤离事件 (evacuation)
- ✅ 撤离记录 (evacuation_record)
- ✅ 人员轨迹 (person_track)
- ✅ 人员人脸 (person_face)
- ✅ 人员步态 (person_gait)

### 2. 隔离机制

- **查询隔离**: 所有查询SQL都添加了 `platform_no` 过滤条件
- **新增隔离**: 新增数据时自动从上下文获取并设置 `platform_no`
- **修改隔离**: 修改时不允许跨平台修改数据
- **删除隔离**: 删除操作只影响当前平台的数据

## 六、注意事项

### 1. 平台切换时机

- 用户登录后应立即选择或切换到目标平台
- 建议在首页或顶部导航栏提供平台选择器
- 切换平台后应刷新当前页面数据

### 2. 默认平台

- 如果未设置平台号，系统可能返回空数据或所有数据
- 建议在前端设置默认平台或强制用户选择平台

### 3. 数据迁移

- 对于已有数据，SQL脚本会将所有数据的 `platform_no` 设置为 `PLATFORM-A01`
- 如果需要分配到不同平台，需要手动更新 `platform_no` 字段

### 4. 性能优化

- 所有 `platform_no` 字段都已添加索引
- 建议在常用查询中将 `platform_no` 作为第一个过滤条件

## 七、常见问题

### Q1: 切换平台后数据没有变化？

**A**: 检查以下几点：
1. 前端是否在请求头中正确携带了 `X-Platform-No`
2. 数据库中对应表是否有 `platform_no` 字段
3. Mapper XML 中是否添加了平台号过滤条件

### Q2: 如何批量修改数据的平台号？

**A**: 使用UPDATE语句：
```sql
UPDATE person SET platform_no = 'PLATFORM-B02' WHERE id IN (1, 2, 3);
```

### Q3: 能否实现跨平台查询？

**A**: 可以，在Mapper XML中去掉 `platform_no` 过滤条件即可，但需要注意权限控制。

### Q4: 平台配置在哪里管理？

**A**: 访问系统菜单中的"钻井平台配置"模块（需要先配置菜单权限）。
